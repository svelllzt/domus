# Domus

Платформа для поиска и пристройства питомцев из приютов.

## О проекте

`Domus` — веб-приложение, в котором:
- пользователи ищут животных по фильтрам и отправляют заявки на знакомство;
- приюты управляют карточками питомцев и обрабатывают заявки;
- глобальный администратор модерирует заявки на регистрацию приютов и управляет SMTP-настройками.

Проект ориентирован на простое локальное развёртывание и подготовлен к деплою.

## Функционал

### Для пользователей
- регистрация и вход;
- подтверждение email кодом из письма (6 цифр);
- восстановление пароля через email;
- поиск питомцев с фильтрами и динамической подгрузкой;
- избранное;
- отправка заявки на знакомство с питомцем;
- личный кабинет с:
  - избранными анкетами,
  - историей заявок,
  - сменой пароля.

### Для приютов
- кабинет приюта;
- создание и управление анкетами животных;
- работа с входящими заявками;
- отдельный сценарий подачи заявки на регистрацию приюта для `shelter_staff`.

### Для глобального администратора
- глобальная админ-панель;
- модерация заявок на регистрацию приютов (approve/reject);
- управление SMTP (сохранение, тест отправки, очистка конфига).

## Техническая часть

- Backend: `FastAPI`
- Templates: `Jinja2`
- База данных: `SQLite`
- Стили/фронтенд: ванильные `CSS` и `JS`
- Пароли: `passlib` (`bcrypt`)
- Токены/сессии: `itsdangerous`
- Шифрование SMTP-пароля: `cryptography` (Fernet)

## Структура проекта

- `main.py` — точка входа приложения
- `config.py` — конфигурация
- `database.py` — подключение к БД и миграции
- `routes/` — роуты (`auth`, `admin`, `shelter`, `user`, `api`, `main_routes`)
- `models/` — SQL-слой (пользователи, питомцы, заявки, избранное и т.д.)
- `services/` — сервисы (email, crypto)
- `templates/` — HTML-шаблоны
- `static/` — CSS/JS/загрузки
- `seed_data.py` — создание и очистка данных (включая тестовые)

## Запуск

### 1) Установить зависимости

```bash
pip install -r requirements.txt
```

### 2) Сгенерировать ключи

Сгенерируйте минимум два разных секрета:
- `PETS_SECRET_KEY` — подпись сессий/токенов;
- `PETS_SMTP_ENCRYPTION_KEY` — шифрование SMTP-пароля в БД.

```bash
python -c "import secrets; print(secrets.token_urlsafe(64))"
```

Запустите команду два раза и используйте разные значения.

### 3) Настроить переменные окружения (минимум)

- `PETS_ENV` (`development`/`production`)
- `PETS_SECRET_KEY`
- `PETS_SMTP_ENCRYPTION_KEY`
- `PETS_APP_BASE_URL`
- `PETS_GLOBAL_ADMIN_EMAIL`
- `PETS_GLOBAL_ADMIN_PASSWORD`

Пример для PowerShell (текущая сессия):

```powershell
$env:PETS_ENV="development"
$env:PETS_SECRET_KEY="...ваш_секрет..."
$env:PETS_SMTP_ENCRYPTION_KEY="...ваш_ключ_шифрования..."
$env:PETS_APP_BASE_URL="http://127.0.0.1:8000"
$env:PETS_GLOBAL_ADMIN_EMAIL="root@domus.local"
$env:PETS_GLOBAL_ADMIN_PASSWORD="your-strong-password"
```

### 4) Инициализация БД и админа

```bash
python seed_data.py
```

### 5) Запуск приложения

```bash
python main.py
```

Основная команда для локального запуска:

```bash
uvicorn main:app --reload
```

Для production используйте запуск без `--reload`.

## Работа с данными

- Очистка данных (с сохранением единственного глобального админа из конфига):
  - функция `clear_demo_data()` в `seed_data.py`
- Добавление тестовых данных:
  - функция `seed_test_data()` в `seed_data.py`

## Безопасность и ограничения

- Ролевые проверки для `user / shelter_staff / shelter_admin / global_admin`;
- ограничения доступа к кабинетам и action-роутам по ролям;
- обязательная верификация email для обычных пользователей;
- `global_admin` синхронизируется из конфига;
- SMTP-пароль не хранится в открытом виде.

## Соли и зачем они нужны

В проекте используются:
- `TOKEN_SALT`
- `EMAIL_TOKEN_SALT`
- `PASSWORD_RESET_SALT`

Соли добавляют дополнительный контекст к подписи токенов и уменьшают риск коллизий между разными типами токенов. В обычной эксплуатации их можно оставить константными внутри проекта, но при миграции/аудите безопасности стоит зафиксировать и хранить их согласованно с окружением.
